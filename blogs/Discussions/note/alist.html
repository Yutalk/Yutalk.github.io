基于webDav的外部网络流媒体服务搭建

v2talk2023-01-02 12:00:00Discussionsnote

# 前言

之前在内网环境搭建过Plex Media Server，但在外网环境下的访问体验不是很好，最主要的原因在于住宅公网动态ip的上行速率太低，且不稳定，视频传输存在严重的卡顿现象。考虑通过webDav协议进行搭建适合外部网络访问的流媒体在线服务

# 开始

# Alist

Alist文档地址：Alistopen in new window

TIP

国内云函数的演变过程的终点都是收费，迫使弃用之前的OneManager项目

前车之鉴，本次部署平台选用 Replitopen in new window，支持https，配合优选CDN实现国内加速访问

# 部署

参考项目文档

  1. 使用Github账户便捷登录 Replitopen in new window
  2. 导入项目 alist-replitopen in new window （点击 Run on Replit 按钮）
  3. 完成项目导入，Run
  4. 项目初次启动会在控制台输出日志信息，包括默认随机密码
  5. 登录后，参考Alist文档配置相关选项

# 配置

  1. 添加Onedrive盘
  2. 挂载路径：/
  3. web代理：开启
  4. webDAV策略：本地代理
  5. 根文件夹路径：/Movie
  6. 其他参数参考说明文档

# infuse配置

  1. infuse需要升级为PRO版本（支持WebDav协议）
  2. 添加WebDav
  3. 通讯协议：WebDav（HTTPS）
  4. 位址：域名，如：mvs.v2talk.cc
  5. 用户名：Alist用户名，默认为：admin
  6. 密码：Alist密码，随机生成，可以在用户菜单栏进行修改
  7. 高级选项下，路径为：/dav
  8. enjoy!

# 总结

CDN加速下通过Replit代理访问世纪互联，基本可以实现流畅播放2G大小的视频文件。

考虑入手pikpak网盘，目前在国内应该可以实现全网速观看，阿里云盘劝退，不在考虑范围。

# 注：Replit的调用规则会在无网络请求是暂时关闭服务，响应请求时重启服务，考虑在服务器上配置自动任务，实现全天候待机
